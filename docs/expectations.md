# General Summary:

i run a software agency where i have people working as middle-men or workers.
the middlemen brings in clients/work for ther workers to do.
the client pays, middlemen records payments, creates invoices, and splits between him and his workers for a project.
any user can either be a middleman or a worker (they could switch accounts using upwork style client and freelancer account)

# user level data security

each user mode i.e. worker or middlemen sees pages and data related to only them
details are in the docs

# client DB

we are keeping a profile for each client. who they are, social contacts, company profile, how many projects did this client give, how many are running etc. it is like our own CRM.

# payment spliting

there should be a payment splitting system in place  which allows flexibility in how people get paid. initial worker-project allocation should be allowed to be overridden for new payments because there could be scenarios where both the party would agree on a change in the allocation mid way.

we should also be keeping full history of playments along with other details such as what was the allocation at a certain time.

payments are being done off paltform, but we should be able to manage how much payment has been done and to whom and how much is due and to whom

# business expenses

we could make business expenses for the agency so that should be recorded somewhere secular but should be able to be added into calculations for net profit etc

# project tasks

i am managing only projects and their payments here. project management and tickets are being made on aseparate platform.

# scale to saas

I am thinking of scaling this into a SaaS later where here could be multiple people managing their own agencies (software or not)
